# DCCN Licensing

**License:** GPL-3.0-or-later
**Model:** Dual-license commercial structure
**System:** Deterministic Convergent Compute Network (DCCN)

---

## Overview

DCCN uses a dual-licensing model:

* **Open-source usage** is available under GPL-3.0-or-later
* **Commercial proprietary usage** requires a paid commercial license

This structure follows the same general licensing strategy used by systems such as:

* EventStoreDB
* Qt
* MySQL
* Redis (BSL-style commercialization)
* Temporal ecosystem monetization models

The intent is:

* preserve open infrastructure access
* prevent uncompensated proprietary enclosure
* fund long-term deterministic infrastructure development
* support enterprise operational requirements

# Dual-License Rationale

GPL-3.0 ensures improvements to the open ecosystem remain open.

Commercial licensing exists to allow:

* proprietary integration
* closed-source deployment
* enterprise compliance operation
* funded maintenance and support

Without a commercial exception, any proprietary system embedding DCCN components would inherit GPL obligations.

The commercial license functions as an explicit exemption from GPL reciprocity requirements.
